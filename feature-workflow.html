Workflow-Stufen — nicht jedes Feature braucht Full Scrum

Stufe 3 — Full Scrum
Repo-übergreifend, komplex. PO → SM → Issues in allen Repos → parallel arbeiten → PO reviewed.
Beispiel: Neues Kampfsystem mit API, GM-Dashboard, Player-View, UE5
Stufe 2 — Lite
Ein Repo, klares Scope. Issue direkt im Repo → Senior plant → Worker implementiert.
Beispiel: Neue Tabelle im Lore-Admin für Kreaturen
Stufe 1 — Bugfix
1-2 Dateien. Issue → Debugger fixt → PR → Merge.
Beispiel: Pagination broken in session-player

Agent-Verteilung über die Repos

Agent Model Lebt in Macht was
Product Owner opusplan orchestrator Specs, Issues, Cross-Repo Review
Scrum Master opusplan orchestrator Sprint-Board, Task-Verteilung
Designer sonnetplan orchestrator UI/UX Konzepte, Design Tokens
Senior Frontend sonnetplan session-gm session-player lore-admin FSD-Architektur, Lite Features
Worker Frontend sonnet session-gm session-player lore-admin Komponenten nach Spec
Senior Backend sonnetplan api shared API-Architektur, Contracts
Worker Backend sonnet api Routes, Services
Senior UE5 sonnetplan ue5 UE5-Architektur, API-Integration
Worker QS sonnet In jedem Code-Repo — Tests (Vitest, Playwright, UE5 Automation)
Debugger sonnet In jedem Code-Repo — Root-Cause-Analyse, minimaler Fix
