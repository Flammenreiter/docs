== Konzept


Über dem statischen UI liegt ein adaptiver Hintergrund-Layer, der drei Dimensionen des aktiven Charakters in Echtzeit reflektiert.


.Die drei Layer
[cols="1,3,3",options="header"]
|===
| Layer | Dimension | Trigger

| _Layer A_ | _Akademie_ (4 Häuser) | Page-Wechsel, Char-Switch
| _Layer B_ | _Rasse / Ancestry_ (19 Rassen) | Char-Switch (dauerhaft)
| _Layer C_ | _Klasse / Pfad_ (12 Klassen) | Aktionen (ephemer, 1-2s)
|===

== Technische Architektur


=== Asset-Pipeline

[cols="2,2,3",options="header"]
|===
| Asset-Typ | Quelle | Storage

| Lottie-Animations | LottieFiles / fal.ai → AE | `shared/src/assets/atmospheric/lottie/`
| Procedural-SVG-Generators | TS-Funktion + Seed (deterministisch pro Char) | `shared/src/lib/atmospheric/generators/`
|===


=== Performance-Budget

* _Idle-State:_ CPU < 2%, GPU < 5%. Animations pausiert oder im 5-10s-Loop.
* _Power-Mode:_ CPU < 10%, GPU < 20%. 1-2s Burst dann zurück auf Idle.
* _Reduced-Motion-Mode:_ Alle Animationen aus, nur statische Hintergründe.
* _Mobile:_ 3-stufiger Detection-Fallback (modern → reduced → static).
* _Bundle-Impact:_ pro Akademie ~30-50 KB gzipped, pro Rasse ~10-20 KB. Lazy-loaded nach aktivem Char.


=== Procedural Generators

  seed: string                              // character.id
  startCorner: 'tl' | 'tr' | 'bl' | 'br'
  depth: number                             // L-system iterations
  length: number                            // total px
}): SVGPathElement[] => {
  // L-System-basiertes Pflanzen-Wachstum
}
----
