==== RLS-Struktur (geplant)


[source,sql]
----
-- Kampagnen-Isolation
CREATE POLICY "Users can access own campaigns"
  ON campaigns
  USING (created_by_user_id = auth.uid());

-- Spieler dürfen nur ihre Charaktere sehen
CREATE POLICY "Players see own characters"
  ON characters
  USING (player_user_id = auth.uid() OR created_by_user_id = auth.uid());

-- GM sieht alles in der Kampagne
CREATE POLICY "GM sees all campaign data"
  ON session_mechanical_log
  USING (campaign_id IN (SELECT id FROM campaigns WHERE created_by_user_id = auth.uid()));
----

== Security Best Practices (geplant)



[cols="1,1",options="header"]
|===
| Aspekt | Maßnahme

| _Secrets Management_ | GitHub Secrets für API Keys, DB Credentials
| _RLS Enforcement_ | Strenge Row-Level Security Policies
| _HTTPS Only_ | Alle Connections encrypted
| _Rate Limiting_ | Pro User, Pro IP
| _CORS Policy_ | Whitelist nur trusted Origins
| _Input Validation_ | Zod Schema Validation überall
| _SQL Injection_ | Parameterized Queries via ORM
| _CSRF Protection_ | SameSite Cookies
| _XSS Prevention_ | Content Security Policy Headers
|===
