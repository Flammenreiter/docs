== Schicht 1: Supabase PostgREST



=== Zweck


Auto-generierter REST API für einfache Read/Write Operationen auf alle Tabellen.


=== Umfang


~70% aller API-Requests laufen über PostgREST:
* Daten-Abrufen (SELECT mit Filtering, Pagination)
* Einfache Inserts (User erstellt neuen Charakter)
* Einfache Updates (Item-Menge im Inventar ändern)
* Deletes (Charakter löschen)


=== Feature


[source,typescript]
----
// GET /rest/v1/characters?campaign_id=eq.abc123&select=*
// Automatically applies RLS
// Returns JSON, supports:
//  - Filtering: ?level=gt.5
//  - Ordering: ?order=name.asc
//  - Pagination: ?limit=20&offset=40
//  - Relationships: ?select=*,organizations(*)
----


=== Limitierungen


* Keine Custom Business Logic
* Keine Transaktionen über mehrere Tabellen hinweg
* Keine komplexen Validierungen (Zod)
* Kein Event Publishing

_Lösung_: Für komplexe Operationen → Hono API.


== Schicht 2: Hono Application Layer



=== Zweck


TypeScript-basierter Business Logic Layer für komplexe Operationen.


=== Warum Hono statt NestJS/.NET?



[cols="1,1,1,1",options="header"]
|===
| Kriterium | Hono | NestJS | .NET

| _Bundle Size_ | 14 KB | 500 KB+ | N/A (Server)
| _Cold Start_ | <100ms | 500ms+ | 1-2s
| _TypeScript_ | Native | Native | C# (kein TS)
| _Zod Integration_ | Built-in | Addon | Addon
| _Edge Compatible_ | Ja | Nein | Nein
| _Learning Curve_ | Flach | Steil | Steil
| _Ideal für_ | Frontend Dev | Enterprise | Enterprise
|===


_Entscheidung_: Hono. Perfekt für Frontend-Entwickler die auch Backend schreiben müssen.

=== EffectProcessor Service (Herz des Systems)



==== Zweck


Zentrale Stelle wo alle mechanischen Modifikatoren eines Charakters gesammelt und berechnet werden.


==== Quellen


Der EffectProcessor liest MechanicalEffects aus:
. Traits (character_trait_levels → trait_spectrum_effects)
. Items (equipped in character_inventory → item_special_abilities)
. Conditions (character_active_effects)
. Wetter (campaign_calendar_state → seasons/weather → mechanical_effects)
. Saison (mechanical_effects auf seasons)
. Gelände (campaign_travel_days → terrain_types → mechanical_effects)
. Buffs/Debuffs (character_active_effects)
. Konsequenzen (character_criminal_record → legal_consequences → mechanical_effects)
. Drachen-Bond (character_dragon_bonds → dragon_trait_levels resonance)


==== Algorithmus


[source,typescript]
----
class EffectProcessor {
  async getModifierStack(characterId: UUID, target: EffectTarget): Promise<ModifierStack> {
    const effects: MechanicalEffect[] = [];

    // Sammle alle Effekte
    effects.push(...await this.getTraitEffects(characterId));
    effects.push(...await this.getItemEffects(characterId));
    effects.push(...await this.getConditionEffects(characterId));
    effects.push(...await this.getEnvironmentEffects(characterId));

    // Filtere nach target
    const relevantEffects = effects.filter(e => e.target === target);

    // Berechne Modifier
    const stack = this.calculateStack(relevantEffects);

    // Erzeuge transparente Beschreibung
    return {
      finalModifier: stack.total,
      components: [
        { label: 'Base Ability', value: +3 },
        { label: 'Proficiency', value: +2 },
        { label: 'Trait (Loyal)', value: +1 },
        { label: 'Rain Penalty', value: -2 },
        { label: 'Dragon Resonance', value: +1 },
      ],
      description: 'Perception +5: Base +3, Proficiency +2, Loyal +1, Rain -2, Dragon +1'
    };
  }

  private calculateStack(effects: MechanicalEffect[]): CalculatedStack {
    // Vorteil/Nachteil hebt sich auf (D&D Standard)
    const advantages = effects.filter(e => e.operation === 'advantage').length;
    const disadvantages = effects.filter(e => e.operation === 'disadvantage').length;
    const netAdvantage = advantages - disadvantages; // kann negativ sein

    // Sorting: höchster Set-Wert gewinnt
    const setValues = effects
      .filter(e => e.operation === 'set')
      .map(e => e.value)
      .sort((a, b) => b - a);

    let base = setValues.length > 0 ? setValues[0] : 0;

    // Dann Adds stapeln
    const adds = effects
      .filter(e => e.operation === 'add')
      .reduce((sum, e) => sum + e.value, 0);

    // Dann Multiplies kombinieren
    const multiplies = effects
      .filter(e => e.operation === 'multiply')
      .reduce((prod, e) => prod * e.value, 1.0);

    const total = (base + adds) * multiplies;

    return {
      base,
      adds,
      multiplies,
      netAdvantage,
      total
    };
  }
}
----

// supabase/functions/session-transcripts/index.ts
Supabase.functions.http('session-transcripts', async (req, context) => {
  const { sessionId } = await req.json();

  // 1. Hole Session Daten
  const session = await db.sessions.findById(sessionId);

  // 2. Lade Audio aus Google Drive (Patreon Integration)
  const audioFile = await drive.files.get(session.craig_audio_path);

  // 3. Transkribiere mit Whisper
  const transcript = await openai.audio.transcriptions.create({
    file: audioFile,
    model: 'whisper-1',
    language: 'de',
    prompt: 'Gaming session, German language, D&D campaign'
  });

  // 4. Zusammenfasse mit Gemini
  const summary = await gemini.generateContent({
    prompt: `Summarize this D&D session:\n${transcript}`,
    generationConfig: {
      maxOutputTokens: 1000,
      temperature: 0.5
    }
  });

  // 5. Speichere in DB
  await db.session_transcript.create({
    session_id: sessionId,
    transcript_text: transcript,
    summary_text: summary,
    source: 'craig',
    processing_status: 'complete'
  });

  return new Response(JSON.stringify({ success: true }), {
    headers: { 'Content-Type': 'application/json' }
  });
});

== Error Handling & Logging



=== Struktur


[source,typescript]
----
// Alle Errors sind strukturiert
class GameError extends Error {
  constructor(
    public code: string,
    public message: string,
    public statusCode: number,
    public context?: Record<string, any>
  ) {
    super(message);
  }
}

// In Hono
app.onError((err, c) => {
  if (err instanceof GameError) {
    return c.json({
      error: err.code,
      message: err.message,
      context: err.context
    }, err.statusCode);
  }

  // Unerwarteter Error
  return c.json({
    error: 'INTERNAL_ERROR',
    message: 'Something went wrong'
  }, 500);
});

== Caching-Strategie



=== In-Memory Caching (Hono)


[source,typescript]
----
const characterCache = new Map<UUID, Character>();
const CACHE_TTL = 5 * 60 * 1000; // 5 Minuten

async function getCachedCharacter(id: UUID): Promise<Character> {
  if (characterCache.has(id)) {
    const cached = characterCache.get(id)!;
    if (Date.now() - cached.cachedAt < CACHE_TTL) {
      return cached;
    }
  }

  const character = await db.characters.findById(id);
  character.cachedAt = Date.now();
  characterCache.set(id, character);
  return character;
}
----
