
= Shared Types & Enums


Zentrale Typ-Definitionen und Enums für das gesamte System.


== MechanicalEffect: Das Universelle System


Jedes Spielelement mit mechanischen Auswirkungen verwendet denselben Typ:

[source,typescript]
----
interface MechanicalEffect {
  target: EffectTarget
  operation: EffectOperation
  value: number
  condition?: EffectCondition
  duration?: EffectDuration
  description: string
}
----

=== operation: EffectOperation


_WIE wird es beeinflusst:_

[source,typescript]
----
type EffectOperation =
  | 'add'           // +3 (stapelbar)
  | 'set'           // = 15 (höchster gewinnt)
  | 'multiply'      // × 1.5 (kombiniert)
  | 'advantage'     // Roll 2d20 take higher
  | 'disadvantage'  // Roll 2d20 take lower
  | 'immunity'      // Keine Effekte dieser Art
  | 'resistance'    // Halbier Schaden
  | 'vulnerability' // Doppel Schaden
  | 'reroll'        // Re-roll failed check
  | 'critical_range_expand' // 19-20 statt 20
----


=== value: number


_WIEVIEL:_

----
add: value = +3
set: value = 15
multiply: value = 1.5 (× 50%)
advantage/disadvantage: value = 1 (boolean flag)
immunity: value = 1 (boolean)
----


=== condition?: EffectCondition


_WANN gilt der Effekt:_

[source,typescript]
----
type EffectCondition =
  // Wetter
  | { type: 'weather', weather_id: UUID, operator: 'during' | 'not_during' }

  // Zeit
  | { type: 'time_of_day', time: 'dawn'|'morning'|'afternoon'|'dusk'|'night'|'deep_night' }
  | { type: 'season', season: 'winter'|'spring'|'summer'|'autumn' }

  // Gelände
  | { type: 'terrain', terrain_id: UUID, operator: 'in' | 'not_in' }

  // Status
  | { type: 'condition', condition_id: UUID, operator: 'has' | 'not_has' }

  // Kampf
  | { type: 'in_combat', value: true | false }
  | { type: 'initiative_order', position: 'first' | 'first_half' | 'second_half' }

  // Beziehungen
  | { type: 'vs_creature_type', creature_type_id: UUID }
  | { type: 'vs_alignment', alignment: string }
  | { type: 'vs_trait_level', spectrum_id: UUID, operator: 'gt'|'lt'|'eq', threshold: number }

  // Charakter-States
  | { type: 'hp_threshold', operator: 'below'|'above', threshold: number }
  | { type: 'spell_slots_available', min_level: number }

  // Logik
  | { type: 'and', conditions: EffectCondition[] }
  | { type: 'or', conditions: EffectCondition[] }
  | { type: 'not', condition: EffectCondition }
----


=== duration?: EffectDuration


_WIE LANGE gilt der Effekt:_

[source,typescript]
----
type EffectDuration =
  // Kampf
  | { type: 'rounds', value: number }              // 3 Runden
  | { type: 'concentration', max_duration?: number } // Solange konzentriert (max 1 Minute)

  // Ruhen
  | { type: 'short_rest' }                         // Bis kurze Rast
  | { type: 'long_rest' }                          // Bis lange Rast

  // Zeit
  | { type: 'hours', value: number }               // 8 Stunden
  | { type: 'days', value: number }                // 1 Tag
  | { type: 'scene' }                              // Diese Szene

  // Auf Ablauf
  | { type: 'until_condition', condition: EffectCondition } // Bis Bedingung erfüllt

  // Permanent
  | { type: 'permanent' }

  // Sofort/Einmalig
  | { type: 'instant' }
----


=== description: string


_UI-Text für Spieler:_

----
"Perception +2 in dim light"
"Gain advantage on Stealth checks"
"Roll damage twice, use higher"
----


== Zod Schemas


Alle Types haben entsprechende Zod Schemas für Validierung:

[source,typescript]
----
import { z } from 'zod';

export const EffectTargetSchema = z.enum([
  'strength',
  'dexterity',
  // ... all 50+ targets
]);

export const EffectOperationSchema = z.enum([
  'add',
  'set',
  'multiply',
  // ...
]);

export const MechanicalEffectSchema = z.object({
  target: EffectTargetSchema,
  operation: EffectOperationSchema,
  value: z.number(),
  condition: z.object({
    type: z.string(),
    // ... conditional fields
  }).optional(),
  duration: z.object({
    type: z.string(),
    // ... duration fields
  }).optional(),
  description: z.string().min(1).max(200),
});
----

== Typ-Generierung aus DB


[source,bash]
----
# CLI command
supabase gen types typescript --project-id $PROJECT_ID > database.types.ts
----

Generiert automatisch TypeScript Interfaces für alle Tabellen und ihre Spalten:

[source,typescript]
----
// Generiert automatisch:
export interface Database {
  public: {
    Tables: {
      characters: {
        Row: {
          id: string
          user_id: string | null
          campaign_id: string
          name: string
          level: number
          hp_current: number
          hp_max: number
          // ... alle Spalten
        }
        Insert: { /* ohne id, created_at */ }
        Update: { /* alle optional */ }
      }
      // ... 200 weitere Tabellen
    }
  }
}
----

Diese generieren Types werden dann in `@flammenreiter/shared-types` handbearbeitet und validiert.


== Best Practices



=== 1. Validierung auf jedem Layer


[source,typescript]
----
// API-Eingabe
const input = CharacterSchema.parse(req.body);

// DB-Query
const character = await db.characters.findById(input.id);

// Output
const response = CharacterSchema.parse(character);
return res.json(response);
----


=== 2. Nie auf Generierte Types verlassen


Die generierten Database Types sind für IDE-Autocomplete, aber für Business Logic nutze explizite Schemas.


=== 3. MechanicalEffect überall nutzen


Nicht: `item.perception_bonus: 2`
Sondern: `item.mechanical_effects: [{ target: 'perception', operation: 'add', value: 2 }]`


=== 4. Enum statt String


[source,typescript]
----
// Gut
const season: Season = 'winter';

// Nicht
const season: string = 'winter';
----
